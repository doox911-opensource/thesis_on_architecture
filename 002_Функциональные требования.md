## Интеграция партнёра

Любая интеграция осуществляется через API приложения и API внешних систем партнёров.

### Интеграция каталогов продуктов партнёра

[Продукты](000_Терминология.md#product) партнёров доступны пользователю в интерфейсах. Пользователь должен иметь возможность включения/отключения отображения [продуктов](000_Терминология.md#product) в интерфейсах приложения после приобретения [премиум подписки](000_Терминология.md#premium_subscription) в [личном кабинете](000_Терминология.md#user_lk). Управление отображением [продуктов](000_Терминология.md#product) должно быть разделено по типам (спортивный инвентарь, промо-акции и т. д.) с возможность отключить все типы продуктов.

## Интеграции в приложении

Приложение должно интегрироваться с системами аутентификации для идентификации пользователя так как к пользователю привязаны:

- Персональные настройки приложения;
- Виды спорта;
- Тренировки;
- Спортивные гаджеты;
- Новостные предпочтения;
- Продуктовые предпочтения.

Приложение должно интегрироваться с текущим сервисом авторизации.

Приложение должно интегрироваться с сервисами уведомлений для возможности уведомления пользователя через рассылки.

Приложение должно интегрироваться с системами по контролю качества контента в приложении для отслеживания эффективность контента, определения наиболее популярных темы и форматов, вести работы цензурного характера, связанные с нарушением авторских прав или несоответствием контента правилам и нормам компании, или законам РФ.

### Система аутентификации

Аутентификация производится сторонними сервисами по технологии SSO (Single Sign-On). Список компаний/сервисов предоставляющих аутентификацию и допустимых для интеграции:

- Google;
- Яндекс;
- VK;
- Госуслуги.

### Система авторизации

Сервис авторизации должен быть совместим со списком доступных сервисов аутентификации.

Сервис авторизации должен разграничивать пользователей согласно ролевой модели(_добавить описание ролевой модели_).

### Сервис пользовательских предпочтений

Сервис пользовательских предпочтений должен предоставлять:

- Настройки приложения;
- Настройки по продуктам при наличии подписки;
- Настройка источников хранения данных приложения.

### Сервис тренировок

Сервис тренировок должен предоставить:

- Возможность выбора вида(ов) спорта для тренировок;
- Возможность проведения тренировки;
- Хранение статистики тренировок;
- Экспорт статистики тренировок;
- Предлагать базовые тренировки по:
  - бег;
  - отжимания;
  - пресс;
  - подтягивания;
  - жим;
  - приседания.

Все тренировки привязаны к пользователю.

### Сервис собственной продукции

Сервис предоставляет список продуктов компании и возможность управления ими.

### Сервис партнёра

Предоставляет информацию о партнёре.

### Сервис продуктов партнёра

Предоставляет доступный список продуктов партнёров.

### Сервис новостных рекомендаций

Сервис предоставляет спортивные новости с конфигурационными возможностями.

Сервис должен предоставлять возможность получать новости обо всех видах спорта или о конкретных видах, выбранных пользователем. Это может быть отдельный вид спорта, по которому пользователь хочет получать новости, или виды спорта, связанные с пользовательскими тренировками.

### Сервис спортивных рекомендаций

Сервис предоставляет рекомендации по персональным тренировкам выбранного вида спорта.

Для формирования рекомендаций сервис использует внутренний алгоритм который опирается на:

- физиологических особенностей пользователя;
- данные о тренировках по конкретному по видам спорта;
- статистики других тренировок пользователей схожих с физиологическими особенностями конкретного пользователя.

### Сервисы уведомлений

В единое приложение должно быть интегрированы разработаны ранее сервисы уведомлений.

### Сервис системы контроля за контентом

Вся информация, которая вносится в приложение пользователем должна проверяться через внешний сервис по контролю качества контента. Сервис предоставляется фирмой - Цензура.РФ

### Сервис мотивации

Сервис мотивации должен стимулировать пользователя чаще заниматься спортом. Стимулирование должно осуществляться благодаря известным цитатам выдающихся спортсменов.

Цитаты выдающихся спортсменов могут быть в формате текста аудио или видео. Цитаты должны выдаваться с учётом языка установленного в приложении.

Если цитата имеет отношение к конкретному виду спорта, то она должна отдаваться именно к конкретному виду спорта.

## Страницы

Страницы реализуется в соответствии с figma.

### Страница входа

Страница входа не должна содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Страница входа должна содержать список сервисов, через которые можно выполнить аутентификацию.

По клику на сервис пользователь должен перенаправляется на страницу аутентификации сервиса аутентификации.

Переход на страницу входа должен осуществляется автоматически если пользователь не авторизован в приложении или по прямой ссылке в случае web-версии.

Если пользователь авторизован при попадании страницу входа пользователь автоматически перенаправляется на страницу [личного кабинета](#личный-кабинет).

### Страница выхода

Страница выхода не реализуется.

Выход из приложения осуществляется через кнопку выхода в [sidebar](#интерфейс-sidebar) с подтверждением действия.

После выхода из приложения пользователь автоматически перебрасывается на [страницу входа](#страница-входа).

## Личный кабинет

Страница личного кабинета должна содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Содержимое должно включать в себя:

- Ссылки на страницы тренировок заказов и подписок;
- Статистику по последней тренировки;
- Создание тренировки;

Переход на страницу личного кабинета должен осуществляться из [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) или по прямой ссылке в случае web-версии.

## Корзина

Страница корзина должна содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Интерфейс корзины в приложении должен стоять из разделов:

- Список продуктов. Состоящий из продуктов, которых пользователь добавил в корзину, с возможностью изменить количество продуктов или удалить его из списка;
- Общая информация, где отображается общая стоимость продуктов в корзине и итоговая стоимость;
- Информация о пользователе, где указываются данные пользователя, такие как имя, адрес, телефон и email;
- Способы оплаты, где представлены доступные способы оплаты (наличный и безналичный) для данного региона;
- Элемент оформления заказа: при нажатии на него заказ отправляется на обработку, после чего с пользователем связывается менеджер для подтверждения и отправки/отмены заказа.

## Каталог продуктов

Страница каталога продуктов должна содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Страница каталога продуктов должна отображать список карточек продукта с минимальным описанием, изображением и элементом добавления в корзину. На карточке продукта должна отображаться информация:

- источник продукта (наш или партнёрский);
- тип продукта (услуга или продукт);
- возможность оплатить [спортиками](000_Терминология.md#sportick).

На карточке продукта должен присутствовать элемент добавления продукта корзину.

На странице продуктов должен быть реализован фильтр по продуктам.

Переход на страницу каталога продуктов должен осуществляться из [sidebar](#интерфейс-sidebar) или по прямой ссылке в случае web-версии.

## Чат

Страница чата должен содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Переход на страницу чата должен осуществляться из [footer](#интерфейс-footer) или по прямой ссылке в случае web-версии.

Содержимое страницы чата - это разработанный ранее чат, который был отдельным приложением.

## Обратная связь

Страница обратный связи должна содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Страница обратной связи должна содержать форму для обратной связи с возможностью ввести заголовок, описание, возможностью приложить файлы.

Переход на страницу обратной связи должен осуществляться из [sidebar](#интерфейс-sidebar) или по прямой ссылке в случае web-версии.

## О приложении

Страница о приложении должна содержать интерфейсы [toolbar](#интерфейс-toolbar), [sidebar](#интерфейс-sidebar) и [footer](#интерфейс-footer).

Страница о приложении должна содержать подробное описание о приложении, его возможностях и версии приложения.

Переход на страницу о приложение должен осуществляться из [sidebar](#интерфейс-sidebar) или по прямой ссылке в случае web-версии.

## Интерфейсы

Интерфейсы реализуется в соответствии с figma.

Реализованные интерфейсы (toolbar, sidebar, footer, поиск) переносятся в единое приложение.

### Интерфейс toolbar

Интерфейс toolbar должен содержать в себе поиск по приложению, элемент перехода на страницу [корзины](#корзина) и в личный кабинет.

### Интерфейс sidebar

Интерфейс sidebar должен содержать список элементов для перехода в:

- [Личный кабинет](#личный-кабинет);
- [Каталог продуктов](#каталог-продуктов);
- [Обратная связь](#обратная-связь);
- [О приложении](#о-приложении).

Интерфейс sidebar должен содержать элемент выхода из приложения.

### Интерфейс footer

Интерфейс footer должен содержать в себе элемент для перехода на страницы:

- [Чат](#чат).

## Внедрение премиум подписки

Приобретение в приложении [премиум подписки](000_Терминология.md#premium_subscription) реализуется в интерфейсе "Подписки" в [личном кабинете](000_Терминология.md#user_lk).

[Премиум подписка](000_Терминология.md#premium_subscription) может приобретаться за рубли или за [спортики](000_Терминология.md#sportick).

[Премиум подписка](000_Терминология.md#premium_subscription) должна предоставлять пользователю дополнительный функционал:

- Включение/отключение [продуктов](000_Терминология.md#product) наших и (или) партнёров;
- Возможность экспорта статистики тренировок;
- Удалённое хранение данных приложения;
- Предоставление скидок на наши [продукты](000_Терминология.md#product) и [продукты](000_Терминология.md#product) партнёров.
